Loop Engineering

The shift

Instead of prompting, build loops that prompt the system.

AdaL’s working definition

A goal-driven system that coordinates multiple agents to a verified outcome — autonomously, without humans in the loop.

Aligned with the industry direction. The exact definition is still emerging — this is how we make the outer-loop claim precise enough to build against.

Why Loops Became Possible

01 · Self-validation

Close the test loop

Browser use and computer use let agents check their own work with evidence — not only claim success.

02 · Multi-agent roles

Fight context rot

Separate Builder and Evaluator roles, isolated context, specialized prompts, independent review.

These make autonomous loops possible. They do not yet make the full human outer loop reliable.

A Different Path to Autonomy

Not this path

A better single agent
  • Still mostly one agent, one trajectory
  • Product dev is multi-stage — plan, build, test, review, iterate
  • Stuffing the outer loop into the worker complicates a well-defined infra scope
  • Coding agents stay great for manual collab — keep them simple

This path · loop engineering

Model what the engineer still does
  • Start from the 3rd layer: the human outer loop
  • Replace that role with another agent — the engineer runtime
  • It models engineer behaviors across the lifecycle
  • It works with coding agents of today — does not replace them

Autonomy rises when we automate the engineer layer — not when we overload a single worker.

When to Use Loops — and When Not

One principle: loop engineering only makes sense when the outcome is clear enough to delegate.

Use loops when

The goal can be verified
  • Concrete outcome, not only an activity
  • Success criteria you can check
  • Scope, constraints, and stop conditions
  • You’re ready to hand off execution

Don’t force loops when

You’re still discovering the goal
  • The idea is unclear or still evolving
  • You want to brainstorm and explore
  • The human should stay deep in the creative loop
  • Today’s agents as collaborators are enough

Good fit: ship a scoped PR to green with a known quality bar.

Bad fit: “Make a billion. Make no mistakes.” — motion without a stop condition.
