Loops of Today

The Claude Code team has published a public guide — Getting started with loops — the closest thing to an official implementation of loop-oriented workflows. It introduces primitives like /goal, /loop, dynamic workflows, custom skills, and a growing collection of building blocks for orchestrating agents.

  • goals and stop conditions — verifiable exit criteria
  • repeated or scheduled work — local intervals and cloud routines
  • dynamic workflows — agents that adapt as they run
  • skills and reusable building blocks — composed across loops
  • worker coordination and evaluation — orchestrating many agents

The user still designs the system: how many workers to spawn, how they communicate, when they hand work off, how they evaluate one another, how they recover from failures, and when to stop the loop.

What Makes Loops Possible

  • Goal-driven work
    A verifiable objective the loop can measure against.
  • Self-validation
    Browser use and computer use close the testing loop — impossible before.
  • Multi-agent roles
    Evaluator and builder — different context, same or different model, specialized prompts (code review).

08 / 11

What Makes Loops Inaccessible

High complexity Reliable loops require coordinating goals, tools, workers, validation, recovery, state, and stop conditions.
High cost More capable models, longer contexts, browser use, repeated validation, and multiple workers compound runtime cost.
